# 자동매매 시스템 구축 가이드

**목표**: RSI(5) 평균회귀 전략을 1,000만원으로 자동 매매

---

## 1. 증권사 추천: 한국투자증권 (1순위)

| 항목 | 내용 |
|------|------|
| **API 방식** | REST API + WebSocket (현대적) |
| **플랫폼** | Mac/Linux/Windows 모두 지원 |
| **Python 라이브러리** | `mojito2` (pip install) |
| **모의투자** | 지원 (별도 App Key 발급) |
| **수수료** | ~0.015% (이벤트 시 0%) |
| **API 제한** | ~20 요청/초 |
| **인증** | OAuth 2.0 (App Key + Secret) |
| **문서** | 최우수 (Swagger 스타일 + 샘플코드) |

### 왜 한국투자증권인가?

1. **Mac에서 바로 사용** - 다른 증권사(키움, 대신)는 Windows 전용
2. **REST API** - 현대적이고 Python 친화적
3. **mojito2** - 깔끔한 Python 래퍼, 활발한 유지보수
4. **모의투자** - 실제 돈 투입 전 충분한 테스트 가능
5. **10만원 이상이면 계좌 개설 가능** - 1,000만원 충분

## 3. 설정 단계 (한국투자증권)

### Step 1: 계좌 개설
1. 한국투자증권 앱 다운로드
2. 비대면 계좌개설 (신분증 + 휴대폰 인증)
3. 1,000만원 입금

### Step 2: API 키 발급
1. https://apiportal.koreainvestment.com 접속
2. 회원가입 → 앱 생성
3. **App Key**, **App Secret** 발급 (약 1 영업일)
4. 모의투자용 키도 별도 발급

### Step 3: 개발환경 설정
```bash
pip install mojito2
```

### Step 4: 모의투자 테스트
```python
import mojito

# 모의투자 브로커 생성
broker = mojito.KoreaInvestment(
    api_key="YOUR_APP_KEY",
    api_secret="YOUR_APP_SECRET",
    acc_no="YOUR_ACCOUNT_NO",
    mock=True  # 모의투자 모드
)

# 현재가 조회
price = broker.fetch_price("005930")  # 삼성전자
print(price)

# 매수 주문
broker.create_market_buy_order("005930", 10)  # 삼성전자 10주 시장가 매수

# 잔고 조회
balance = broker.fetch_balance()
print(balance)
```

### Step 5: 실전 전환
```python
# mock=False로 변경하면 실전 매매
broker = mojito.KoreaInvestment(
    api_key="YOUR_REAL_APP_KEY",
    api_secret="YOUR_REAL_APP_SECRET",
    acc_no="YOUR_ACCOUNT_NO",
    mock=False  # 실전!
)
```

## 4. 자동매매 봇 구조

```
auto_trader/
├── config.py           # API 키, 계좌번호 (환경변수로 관리)
├── broker.py           # 한국투자증권 API 래퍼
├── signal_generator.py # RSI(5) 전략 시그널 생성
├── trader.py           # 자동 매매 로직 (매수/매도/손절)
├── risk_manager.py     # 포지션 관리, 리스크 제한
├── scheduler.py        # 정기 실행 스케줄러 (장 마감 전 실행)
├── logger.py           # 거래 기록 로깅
└── main.py             # 메인 실행
```

### 매매 흐름

```
매일 15:00 (장 마감 30분 전)
  │
  ├─ 1) 전 종목 현재가 + 지표 계산
  │     └─ RSI(5), SMA(100), SMA(3), ATR(14) 계산
  │
  ├─ 2) 보유 종목 청산 조건 확인
  │     ├─ 종가 > SMA(3)? → 매도 (시그널 청산)
  │     ├─ 손절가 이탈? → 매도 (손절)
  │     └─ 3일 경과? → 매도 (시간 청산)
  │
  ├─ 3) 신규 매수 조건 확인
  │     ├─ RSI(5) < 10?
  │     ├─ 종가 > SMA(100)?
  │     └─ 포지션 한도 미달? → 매수
  │
  └─ 4) 주문 실행 + 로깅
```

## 5. 1,000만원 포지션 관리

| 항목 | 설정 |
|------|------|
| **총 자본** | 10,000,000원 |
| **종목당 최대** | 2,000,000원 (20%) |
| **동시 보유** | 최대 5종목 |
| **종목당 리스크** | 100,000원 (1%) |
| **손절 기준** | ATR(14) × 2.5 |

### 매수 수량 계산 예시
```
삼성전자 현재가: 60,000원
ATR(14): 2,000원
손절가: 60,000 - (2,000 × 2.5) = 55,000원
리스크/주: 5,000원
최대 리스크: 100,000원 (1%)
매수 수량: 100,000 / 5,000 = 20주
투자금: 20 × 60,000 = 1,200,000원
```

## 6. 법적/세금 사항

| 항목 | 내용 |
|------|------|
| **자동매매 허용** | ✅ 개인 투자자 제한 없음 |
| **거래세** | 매도 시 ~0.18% (자동 원천징수) |
| **양도소득세** | 대주주(10억+) 아니면 비과세 |
| **배당세** | 15.4% 원천징수 |
| **금지 행위** | 시세조종, 허수 주문, 스푸핑 |

## 7. 주의사항

⚠️ **반드시 모의투자로 최소 1개월 테스트 후 실전 전환**

- API 키는 절대 코드에 하드코딩하지 말 것 (환경변수 사용)
- 토큰은 약 24시간마다 갱신 필요
- 장 시간 외 주문은 예약 주문으로 처리
- 네트워크 오류, API 장애 대비 예외처리 필수
- 일일 손실 한도 설정 (예: -3% 도달 시 당일 매매 중단)
- 거래 로그 반드시 저장 (추후 분석 + 세무 증빙용)
